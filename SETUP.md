# Edventara Backend Setup Guide

## Database Setup

### 1. Create the Database
Open MySQL and run:
```sql
CREATE DATABASE edventara;
```

### 2. Run the Schema
Run the schema file to create all tables:
```bash
mysql -u root -p edventara < db/schema.sql
```

## Your Registration System

### 📝 Two Registration Flows:

#### **1. SchoolRegistrationPage.jsx** (New School Sign-up)
- School fills out: name, curriculum, country, contact info
- Saves to `schools` table with status **'pending'**
- Admin reviews and approves later
- Shows success modal with "What happens next?"

#### **2. RegisterPage.jsx** (User/Coordinator Registration)
- Requires an **invite code** (generated by school admin)
- Fills out: name, email, password
- Saves to `users` table, awaits admin approval
- After approval, can login to coordinator dashboard

---

## Database Tables

### `schools` table
- Stores pending school registration requests
- Status: `pending` → `approved` → `rejected`

### `users` table
- Stores coordinators, admins, staff
- Requires `school_id` + valid invite code
- `is_approved` must be TRUE before login allowed

### `invite_codes` table
- Admin generates codes to invite coordinators
- Each code has: school_id, role_assigned, max_uses, expires_at
- Increments `times_used` when coordinator registers

```sql
SELECT * FROM schools;
```

## API Endpoints

### School Registration
```
POST /api/school/register
Body: {
  schoolName: "Test School",
  curriculumType: "International Baccalaureate (IB)",
  country: "United States",
  contactName: "John Doe",
  contactEmail: "john@school.edu",
  estimatedSize: "500 students",
  message: "We'd like to use Edventara..."
}
```

### User Registration (with invite code)
```
POST /api/auth/register
Body: {
  first_name: "John",
  last_name: "Doe",
  email: "john@school.edu",
  password: "securePassword123",
  invite_code: "INVITE2026"
}
```

### User Login
```
POST /api/auth/login
Body: {
  email: "john@school.edu",
  password: "securePassword123",
  school_id: 1
}
Response: { token: "jwt_token", user: {...} }
```
